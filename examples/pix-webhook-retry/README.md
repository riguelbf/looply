# Example: pix-webhook-retry

Este exemplo mostra uma feature pequena atravessando o fluxo `idea-to-prd -> prd-to-stories -> story-to-production`.

## Goal

Demonstrar como o `looply` organiza discovery, planning e delivery em um projeto existente, com artefatos persistidos em Markdown.

## Files

- `discovery/requirement-brief.md`
- `discovery/prd.md`
- `planning/story-backlog.md`
- `planning/stories/story-01-retry-automatico.md`
- `delivery/tech-spec.md`
- `delivery/adr.md`
- `delivery/implementation-summary.md`
- `delivery/review-report.md`
- `delivery/release-plan.md`
- `state/workflow-status.md`
