# Slash Commands

Referencia gerada a partir dos workflows publicados para os hosts suportados.

## [/looply:cloud-workload-design](./slash-commands/looply:cloud-workload-design)

Define cloud topology, governance controls and cost posture for a workload change

- workflow: `cloud-workload-design`
- hosts: `codex`, `claude`

## [/looply:idea-to-prd](./slash-commands/looply:idea-to-prd)

Start discovery and consolidate an idea into an approved PRD

- workflow: `idea-to-prd`
- hosts: `codex`, `claude`

## [/looply:platform-foundation-evolution](./slash-commands/looply:platform-foundation-evolution)

Evolve shared platform foundation with guardrails, governance and cost review

- workflow: `platform-foundation-evolution`
- hosts: `codex`, `claude`

## [/looply:prd-to-stories](./slash-commands/looply:prd-to-stories)

Break an approved PRD into delivery-ready stories

- workflow: `prd-to-stories`
- hosts: `codex`, `claude`

## [/looply:story-to-production](./slash-commands/looply:story-to-production)

Execute delivery for a single approved story until release planning

- workflow: `story-to-production`
- hosts: `codex`, `claude`

## [/looply:workflow-status](./slash-commands/looply:workflow-status)

Inspect or resume the current status of a looply feature workflow

- workflow: `workflow-status`
- hosts: `codex`, `claude`
- aliases: `/looply:resume`, `/looply:next`
