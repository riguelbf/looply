# Code Review Checklist

- [ ] aderencia arquitetural
- [ ] fronteiras de modulo preservadas
- [ ] validacao de entrada explicita
- [ ] contrato e compatibilidade avaliados
- [ ] tratamento de erro
- [ ] acessibilidade e responsividade avaliadas quando houver UI
- [ ] estado local, server state e estado global usados com criterio
- [ ] impacto em persistencia, migracao e rollout avaliado
- [ ] performance e bundle impact avaliados quando houver frontend
- [ ] observabilidade suficiente para diagnostico
- [ ] cobertura de testes relevante
- [ ] riscos destacados
- [ ] documentacao atualizada quando necessario
